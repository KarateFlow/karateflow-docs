\begin{frame}{4. Contesto e Obiettivi del Corso di Software Testing}
  Il progetto KarateFlow, sviluppato inizialmente come evoluzione per il corso di \textbf{SPME}, viene integrato con una robusta suite di test e automazione per il corso di \textbf{Software Testing}.
  \vspace{0.4cm}
  
  I tre obiettivi principali del progetto di Software Testing sono:
  \vspace{0.2cm}
  \begin{enumerate}
    \item \textbf{Integrazione di Testing Avanzato:} Mutation Testing, Performance & Load Testing, Automated E2E Testing.
    \item \textbf{Aumento della Coverage e Riduzione del Debito Tecnico:} Incremento della copertura dei branch ed eliminazione sistematica delle anomalie individuate dai linter e SonarQube.
    \item \textbf{Integrazione di GitHub Actions:} Realizzazione di pipeline in cloud e confronto con l'attuale strumento di CI/CD locale \textbf{Jenkins}.
  \end{enumerate}
\end{frame}
