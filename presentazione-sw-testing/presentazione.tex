\begin{frame}{2. Contesto e Obiettivi del Corso di Software Testing}
  Il progetto KarateFlow, sviluppato inizialmente come evoluzione per il corso di \textbf{SPME}, viene integrato con una robusta suite di test e automazione per il corso di \textbf{Software Testing}.
  \vspace{0.4cm}
  
  I tre obiettivi principali del progetto di Software Testing sono:
  \vspace{0.2cm}
  \begin{enumerate}
    \item \textbf{Integrazione di Testing Avanzato:} Sviluppo di test di integrazione sul backend tramite \texttt{Testcontainers} e modernizzazione del testing del frontend con \texttt{Vitest}.
    \item \textbf{Aumento della Coverage e Riduzione del Debito Tecnico:} Incremento della copertura dei branch ed eliminazione sistematica delle anomalie individuate dai linter e sensori di qualità.
    \item \textbf{Integrazione di GitHub Actions:} Realizzazione di pipeline in cloud e confronto comparativo con l'attuale orchestratore locale \textbf{Jenkins}.
  \end{enumerate}
\end{frame}

\begin{frame}{4. Stato di Partenza: Strategia di Testing}
  La convalida del software all'avvio del progetto presenta la seguente struttura di test di base:
  \vspace{0.3cm}
  \begin{table}[]
    \scriptsize
    \centering
    \begin{tabular}{llll}
      \toprule
      \textbf{Ambito} & \textbf{Tipologia di Test} & \textbf{Tecnologia / Libreria} & \textbf{Scopo principale} \\ \midrule
      \textbf{Backend} & Unit Testing & JUnit 5, Mockito \& MockMvc & Logica dei casi d'uso e REST Controllers. \\
      & Analisi Statica & PMD & Verifica preliminare di code smell e convenzioni. \\
      & Code Coverage & JaCoCo & Rilevamento quantitativo della copertura. \\ \midrule
      \textbf{Frontend} & Unit / UI Testing & Vitest, jsdom \& TestBed & Test isolati di componenti Angular, direttive e servizi. \\
      & Analisi Statica & ESLint \& Prettier & Conformità dello stile TypeScript/Angular. \\
      & Code Coverage & @vitest/coverage-v8 & Analisi quantitativa dei rami UI. \\ \bottomrule
    \end{tabular}
  \end{table}
\end{frame}

\begin{frame}{9. Obiettivo 3: CI/CD e Integrazione GitHub Actions}
  \begin{columns}
    \begin{column}{0.5\textwidth}
      \textbf{Stato Iniziale (Jenkins):}
      \begin{itemize}
        \item Orchestratore locale Jenkins in Docker Compose.
        \item Limite di NAT domestico: trigger forzato tramite polling (\textbf{Poll SCM} ogni 5 min).
      \end{itemize}
      \vspace{0.2cm}
      \textbf{Evoluzione (GitHub Actions):}
      \begin{itemize}
        \item Sviluppo di workflow YAML in cloud per build, test e scansione di qualità.
        \item Trigger nativo istantaneo su Push e Pull Request tramite webhook.
      \end{itemize}

\begin{frame}{12. Sviluppi Futuri del Testing}
  A completamento degli obiettivi principali del corso, le prospettive future includono:
  \vspace{0.3cm}
  \begin{columns}
    \begin{column}{0.48\textwidth}
      \begin{block}{1. Test E2E con Playwright / Cypress}
        Scrittura di scenari completi che validano il flusso utente reale (Login $\rightarrow$ Registrazione test $\rightarrow$ Generazione PDF client-side).
      \end{block}
      \begin{block}{2. Contract Testing con Pact}
        Introduzione di test di contratto per assicurare l'allineamento dei DTO frontend/backend, evitando incompatibilità silenziose.
      \end{block}
    \end{column}
    \begin{column}{0.48\textwidth}
      \begin{block}{3. Dependency \& Vulnerability Scan}
        Automazione di \texttt{dependency-check-maven} (BE) e \texttt{npm audit} (FE) nella pipeline CI per monitorare le vulnerabilità (CVE).
      \end{block}
    \end{column}
  \end{columns}
