\begin{frame}{4. Hosting e Vincoli Fisici}
  \begin{columns}
    \begin{column}{0.6\textwidth}
      \textbf{L'Hosting su Raspberry Pi 5:}
      \begin{itemize}
        \item Consumi ridotti al minimo per azzerare i costi dell'hosting in cloud.
        \item Ideale per una piccola realtà sportiva locale.
        \item \alert{I Vincoli Hardware:} Risorse RAM e CPU estremamente limitate.
      \end{itemize}
      \vspace{0.3cm}
      \textbf{Trade-off e Ottimizzazioni di Stack:}
      \begin{itemize}
        \item Compilazione statica in Angular (servita via Nginx) per minimizzare la memoria RAM occupata in esecuzione.
        \item Uso di container leggeri privi di componenti non necessari.
        \item Resource limits configurati rigidamente per evitare memory leak distruttivi per l'host.
      \end{itemize}
    \end{column}
    \begin{column}{0.35\textwidth}
      \begin{tcolorbox}[colback=karateDark!5, colframe=karateDark, title=Specifiche Host]

        \textbf{RAM:} 8 GB\\

\begin{frame}{7. Uno Sguardo alle Repository di Backend e Frontend}
  La struttura delle repository è stata pensata per massimizzare la leggibilità e ottimizzare il contesto dei modelli di IA. 
  \vspace{0.3cm}
  \begin{columns}

        \begin{itemize}
          \item \textbf{Architettura Esagonale (Ports \& Adapters):}
          \item Core Domain (\emph{model} e \emph{port}) slegato da librerie esterne.
          \item Logica applicativa orchestrata da \emph{usecase}.
          \item Persistenza (MongoDB Adapter) e Controller Web come \emph{adattatori} esterni intercambiabili.
          \item Alta testabilità con test unitari e d'integrazione indipendenti.
        \end{itemize}
      \end{block}
    \end{column}
    \begin{column}{0.48\textwidth}
      \begin{block}{Frontend: Angular (\texttt{karateflow-fe})}
        \begin{itemize}
          \item Single Page Application organizzata in moduli feature.
          \item Utilizzo di componenti riutilizzabili e puliti, privi di logica di business.
          \item Separazione netta tra UI e Business Logic. Posso cambiare totalmente interfaccia senza "rompere" l'applicazione.
          \item Compilazione statica e servita tramite Nginx per minimizzare il consumo di RAM del server.

      \end{block}

  \end{columns}

% Slide 15: Retrosettiva Fase 3: Difficoltà e Soluzioni
\begin{frame}{10. Retrosettiva Fase 3: Difficoltà e Soluzioni}
  Durante la fase di sviluppo core (MVP) sono emerse alcune inefficienze operative:
  \vspace{0.3cm}
  \begin{columns}
    \begin{column}{0.48\textwidth}
      \begin{block}{Le Difficoltà Riscontrate}
        \begin{itemize}
          \item \textbf{Story Mismatch / Scope Creep:} Alcune User Story si sono allargate a dismisura e hanno superato il loro scope, a causa di pipeline CI/CD configurate male o incomplete che ritardavano il feedback.
          \item \textbf{Inconsistenza dei Task:} I nomi dei task e dei commit su Git non erano uniformi perché gestiti manualmente a mente sul portale.
          \item \textbf{Inconsistenza delle PR:} Inizialmente scrivevo ogni PR manualmente e la struttura delle varie PR era inconsistente.
        \end{itemize}
      \end{block}
    \end{column}
    \begin{column}{0.48\textwidth}
      \begin{block}{Le Soluzioni Introdotte}
        \begin{itemize}
          \item \textbf{Issue Templates:} Standardizzazione rigida della creazione di Task, Bug e Requisiti in formato BDD.
          \item \textbf{Pull Request Templates:} Standardizzazione del formato di una PR con descrizione delle modifiche, test eseguiti, possibile immagine se prevista e sezione di tracciabilità con chiusura automatica del TASK associato.
          \item \textbf{Integrazione Branch-Issue:} I branch di sviluppo vengono eliminati una volta effettuata la PR.
        \end{itemize}
      \end{block}
    \end{column}
  \end{columns}
\end{frame}

      \end{block}
      \begin{block}{Qualità del Codice (SonarQube)}

    \begin{column}{0.48\textwidth}
      \begin{block}{Monitoraggio \& Log (Stack PLG)}
        \begin{itemize}

          \item Centralizzazione dei log dei pod di backend, frontend e database per semplificare il debug in produzione.
        \end{itemize}
      \end{block}
      \begin{block}{Miglioramenti UI/UX}
        \begin{itemize}
